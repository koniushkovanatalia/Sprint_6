
### UI тесты для учебного сервиса «Яндекс.Самокат»  
#### Это сервис для аренды самокатов

### Список реализованных тестов:

#### **Главная страница:** 

- Проверка FAQ: текст ответа на вопрос совпадает с ожидаемым

#### **Заказ самоката:**

- Заказ самоката через кнопку "Заказать" вверху страницы
- Заказ самоката через кнопку "Заказать" внизу страницы


#### **Переходы по логотипу "Яндекс" и "Самокат":**

- Проверка перехода на главную страницу Самоката по клику на логотип "Самокат"
- Проверка открытия в новом окне через редирект главной страницы Дзена по клику на логотип "Яндекс"

### Требования для запуска тестов:

- Python 3.x 
- Браузеры: Mozilla Firefox
- WebDriver для браузера (важно, чтобы версия WebDriver соответствовала версии браузера)
- Python-библиотеки: Selenium (для работы с браузером), Pytest (для запуска тестов), Allure (для отчетов)

### Особенности:

- Структура страниц (Page Object Model): все действия с элементами обернуты в классы страниц в директории `pages`. Каждый класс страницы содержит методы для взаимодействия с элементами и проверки их состояния.
- Локаторы: все элементы находятся через локаторы в директории `locators`, что упрощает поддержку тестов и изменение селекторов.
- Тестовые данные: тестовые данные хранятся в файле `data.py`, что позволяет легко изменять входные значения и параметризовать тесты.
- Фикстуры: общие фикстуры для тестов находятся в `conftest.py`, включая инициализацию WebDriver, закрытие браузера и подготовку окружения.
- Отчеты: генерация отчетов через Allure. Результаты тестов сохраняются в `allure-results`, из которых можно создать `HTML-отчет (allure-report)` для наглядного анализа прогонов.
- Параметризация и аннотации: тесты параметризованы с использованием `pytest.mark.parametrize`, а шаги оборачиваются в `@allure.step` для удобного отображения в отчетах.
- Совместимость: тесты проверены в браузерах Chrome и Firefox с учетом различий в рендеринге и поведении элементов.
